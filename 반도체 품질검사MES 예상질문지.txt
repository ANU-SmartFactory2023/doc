2. 어떻게 팀 간의 협업 및 업무분장이 이루어졌고, 어떤 도구를 사용했나요
- 각 공정과 공정에서 사용하는 클래스를 하나 씩 맡아서 함수호출만 하여, 바로 사용할 수 있도록 하였고, 클래스에 문제가 있을 시 소스코드를 짠 사람이 바로 A/S를 하게 하였고, TortoiseGit을 활용하여 shareFolder를 만들어 pull과push를 이용

3. 주요 장애 사례와 그에 대한 대응 및 해결 과정을 설명해주세요
4. 통신방식에 대해 설명해주세요

5. http 통신에 대해 설명해주세요
- 텍스트 기반의 통신 규약으로 인터넷에서 데이터를 주고받을 수 있는 프로토콜

6. http 통신을 선택한 이유가 무엇인가요
- 서버 확장성이 높고, 무상태는 응답 서버를 쉽게 바꿀 수 있기 때문에 무한한 서버 증설이 가능하기 때문에

7. 서버는 무엇을 담당하나요 - 꼬리질문
8. 불량품 추적 및 분석을 어떻게 하나요
- 적외선 센서로 웨이퍼를 정지 시킨 뒤, 각 공정의 이미지센서, 초음파센서, 릴레이센서, 조도센서를 이용하여 불량품을 분석

9. UI는 무엇을 기준으로 제작하였나요
10. UI는 어떤 기능을 가지고 있나요
11. 실시간 웨이퍼상태 및 공정데이터를 처리하고 응답하는데 어떤 방법을 사용하나요
12. 데이터베이스 설계 기준은 어떻게 되나요 
13. DB의 기능은 무엇이 있나요

14. 포토공정,식각공정,EDS공정,EUV공정이 무엇인가요
- 포토공정 : 웨이퍼 위에 반도체 회로를 그려 넣는 과정
- 식각공정 : 반도체의 구조를 형성하는 패턴을 만드는 과정
- EDS공정 : 원하는 품질 수준에 도달했는지를 확인하는 공정
- EUV공정 : 웨이퍼의 노광작업 상태를 반사광으로 확인하는 공정

15. 각 공정 센서들은 무엇을 기준으로 선정하셨나요
16. 라즈베리파이와 각 센서를 통합하고 제어하는 과정에서 어떤 어려움을 겪었나요
17. 라즈베리파이를 선택한 이유와 해당 플랫폼의 장점은 무엇이었나요


18. 라즈베리파이는 왜 2개만 사용하셨나요
- 원래는 각공정에 하나씩 4개의 라즈베리파이를 사용하여, 컨베이어벨트에 라즈베리파이를 두개씩 연결해서 제어를 하려고 했으나, DC모터 구동하는 부분에서 충돌이 일어나서 컨베이어벨트 하나당 하나의 라즈베리파이를 사용하게 되어서 2개를 사용했습니다.

19. 불량품은 어떻게 처리하나요
- 각 공정에서 잡은 센서 값을 기준으로 양품이면 서보모터 각도를 180도로 돌리고, 컨베이어벨트를 돌려서 다음공정으로 보내고 불량품이면 서보모터 각도를 135도로 돌리고, 컨베이어벨트를 돌려서 불량품 바구니로 떨어트리는 방식으로 처리
 
20. 시스템이 어떻게 자동화된 결정을 내리고, 예외 상황에 어떻게 대응하는지 설명해 주세요
21. 등급은 어떻게 분류되나요
